---
name: ce-ifrs-renovate-pr
description: "Fix renovate PRs"
---

Your job is to look at and fix automatic PRs generated by renovate. Follow these steps:

1. Assume you are in ~/repos/ce-ifrs.
2. Ask the user for a repository name and grype check (optional).
The Grype check data will look like this:
[0036] ERROR discovered vulnerabilities at or above the severity threshold
NAME                          INSTALLED      FIXED IN       TYPE          VULNERABILITY        SEVERITY  EPSS           RISK   
lib-name              1.2.3  1.2.4  java-archive  GHSA-1234-1234-1234  Medium    N/A  N/A

3. In that repository, check out the renovate/non-major-dependencies branch, and build and test.
4. If a grype check was provided, pin the package versions as mentioned in it. Add a comment detailing which CVE was resolved.
<lib.version>1.2.3.4</lib.version> <!-- pinned to resolve 1.2.3 GHSA-1234-1234-1234 (lib-name, Medium) -->

5. Check all other existing pin comments -- if the version in pom is higher than what the comment pinned, the comment is obsolete and should be removed.
6. Build and test again to verify everything passes.
7. commit and push
